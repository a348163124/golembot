# Grok Build Engine

The Grok engine invokes the [Grok Build](https://grok.com) CLI (`grok`), xAI's coding agent with headless streaming support.

## Prerequisites

- Install **Grok Build** and ensure `grok` is on your `PATH` (typical install: `~/.grok/bin/grok`)
- Authenticate (choose one):
  - **Browser / device login** — `grok login` (credentials in `~/.grok/auth.json`)
  - **API key** — set `XAI_API_KEY` (from [console.x.ai](https://console.x.ai))

## Configuration

```yaml
# golem.yaml
name: my-bot
engine: grok
# model: grok-4.5   # optional; omit to use Grok's default
```

## Authentication

### Login (interactive)

```bash
grok login
```

GolemBot reuses cached credentials under `~/.grok/auth.json`.

### API key (CI / headless)

```bash
export XAI_API_KEY=xai-...
```

Or via `createAssistant()` / provider block:

```typescript
const bot = createAssistant({ dir: './my-bot', engine: 'grok', apiKey: process.env.XAI_API_KEY })
```

```yaml
engine: grok
provider:
  apiKey: ${XAI_API_KEY}
```

## How GolemBot Invokes Grok

```bash
grok -p "<prompt>" \
  --output-format streaming-json \
  --cwd <workspace> \
  --always-approve \
  [--resume <sessionId>] \
  [-m <model>]
```

| Concern | Behavior |
|---------|----------|
| Output | NDJSON `streaming-json` → unified `StreamEvent` |
| Skills | Symlinked into `.grok/skills/` |
| Rules | Project `AGENTS.md` (generated by GolemBot) |
| Permissions | `--always-approve` by default; set `skipPermissions: false` to require approval |
| MCP | Written to project `.grok/config.toml` from `golem.yaml` `mcp` |
| Session resume | `--resume <sessionId>` from prior `done` event |

### Stream events

| Grok line | GolemBot `StreamEvent` |
|-----------|------------------------|
| `{ type: "text", data }` | `text` |
| `{ type: "thought", data }` | ignored (keeps IM replies clean) |
| `{ type: "end", sessionId }` | `done` |
| `{ type: "error", message }` | `error` |

Tool call events are mapped when present; headless `streaming-json` may not always emit them.

## Choosing a Model

```bash
grok models
```

Common IDs (subject to change): `grok-4.5` (default), `grok-composer-2.5-fast`.

```yaml
engine: grok
model: grok-4.5
```

## Doctor

```bash
golembot doctor
```

Checks that `grok` is on `PATH` and that either `XAI_API_KEY` or `~/.grok/auth.json` is available.

## Notes

- Unattended IM/gateway use should keep auto-approve enabled (default).
- Project `.grok/` is added to `.gitignore` on `init` when `engine: grok`.
- For richer ACP tool streaming, Grok also offers `grok agent stdio`; GolemBot currently uses the simpler headless `-p` path shared with other engines.
